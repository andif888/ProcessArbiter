Using ProcessArbiter  

ProcessArbiter Status Client

 

The image below shows ProcessArbiter Status Client.

Here you can check the status and start and stop the service.
When you run the Status Client the first time after installation you are prompted to enter your license details or automatically request an evaluation license. After that you can start the service and your configuration is done. By default the service is configured to run automatically when windows starts.

This means, the Status Client is more or less for information purpose and to see what's going on under the hood. ProcessArbiter is simply doing its job fully automated without any complicated configuration you have to care about. This is one of the big benefits of ProcessArbiter

 

 

Test ProcessArbiter by generating CPU load

ProcessArbiter comes with a little tool which allows you to to put some load on your CPU and to see how all the stuff works.
 

 

CPU Load test with ProcessArbiter disabled

  1. Make sure ProcessArbiter Service is stopped.
  2. Run Windows Task Manager to see the current CPU utilization.
  3. Click on Action > Run new instance of CPU Resource Hog menu to start the CPU Resource Hog application.
  4. Put some load on your CPU by clicking Start button in CPU Resource Hog.
    (To put more load on the CPU you can run multible instances of CPU Resource Hog simultaniously.)
  5. While your CPU gets exhausted, try to start any other application. For example Internet Explorer, Mozilla Firefox or your favorite email application.
  6. Notice the time, how long it takes until your application is up and running.
  7. Now do the same test while ProcessArbiter is enabled. The steps are outlined below.

 

CPU Load test with ProcessArbiter enabled

  1. Make sure ProcessArbiter Service is started.
  2. Run Windows Task Manager to see the current CPU utilization.
  3. Click on Action > Run new instance of CPU Resource Hog menu to start the CPU Resource Hog application.
  4. Put some load on your CPU by clicking Start button in CPU Resource Hog.
    (To put more load on the CPU you can run multible instances of CPU Resource Hog simultaniously.)
  5. While your CPU gets exhausted, try to start any other application. For example Internet Explorer, Mozilla Firefox or your favorite email application.
  6. Notice the time, how long it takes until your application is up and running.
  7. Now you will notice a significant faster load of your application in this test when ProcessArbiter is enabled.
